Remarks

A call to CreateClient(ProxyHttpClientContext) can return either a new HttpMessageInvoker instance or an old one if the configuration has not changed. If the old configuration is null, a new HttpMessageInvoker is always created. The returned HttpMessageInvoker instance MUST NOT be disposed because it can be used concurrently by several in-flight requests.
